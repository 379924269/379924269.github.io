# 📖 电商数据库字段解释

### PMS_SKU_STOCK (SKU库存表)

| 字段名 | 类型 | 说明 | 示例 |
|--------|------|------|------|
| `id` | bigint | 主键ID | 1 |
| `product_id` | bigint | 商品ID | 1 |
| `sku_code` | varchar | SKU编码 | SKU123456 |
| `price` | decimal | 价格 | 7999.00 |
| `stock` | int | 库存 | 100 |
| `low_stock` | int | 低库存预警值 | 10 |
| `pic` | varchar | 图片URL | /images/iphone-blue.jpg |
| `sale` | int | 销量 | 50 |
| `sp_data` | varchar | 规格数据JSON | [{"key":"颜色","value":"蓝色"}] |
| `lock_stock` | int | 锁定库存 | 0 |

#### 📦 库存字段详解

##### `stock`（总库存）
- **含义**：商品的实际总库存数量
- **作用**：表示仓库中该 SKU 的总数量
- **特点**：是库存的"上限"，不会超过这个值

##### `lock_stock`（锁定库存）
- **含义**：已被订单占用但尚未扣减的库存
- **作用**：临时占用的库存，等待订单确认或支付
- **特点**：是库存的"预留"部分

##### 📊 库存关系图
```
总库存 (stock) = 可用库存 + 锁定库存 (lock_stock)
                    ↓
            可用库存 = stock - lock_stock
```

##### 🛒 业务场景示例

| 阶段 | stock | lock_stock | 可用库存 | 说明 |
|------|-------|-----------|---------|------|
| 初始 | 100 | 0 | 100 | 商品上架，库存充足 |
| 用户下单 | 100 | 10 | 90 | 锁定10件，等待支付 |
| 支付成功 | 90 | 0 | 90 | 扣减库存，释放锁定 |
| 支付超时 | 100 | 0 | 100 | 释放锁定，恢复库存 |

##### ❓ 为什么需要锁定库存？

1. **防止超卖**：用户下单时立即锁定，避免多人同时购买同一件商品
2. **订单流程**：从下单到支付有时间间隔，需要预留库存
3. **库存回滚**：支付失败或订单取消时，可以释放锁定库存

##### 🔒 SQL 中的乐观锁逻辑

```sql
-- 只有当：锁定库存 + 购买数量 <= 总库存 时，才允许锁定
WHERE lock_stock + #{quantity} <= stock
```

这确保了：
- **stock** 是总库存上限
- **lock_stock** 不能超过 **stock**
- 可用库存 = `stock - lock_stock` 始终 ≥ 0

这种设计是电商系统库存管理的标准做法，能有效平衡库存准确性和用户体验.
